# AGENTS.md — руководство для агента

Онбординг для ИИ-агента (и человека), продолжающего работу над проектом. Читается первым.

## Что это

**Gen** — браузерная тактическая RPG: пошаговые бои на квадратной сетке, мета-прогрессия
**Memento Mori**, экспедиции, магазин/таверна, экипировка, моды. Реализация клона по
нормативной спецификации [`docs/spec.md`](docs/spec.md).

**Статус: функционально завершено.** Спека закрыта, обязательного не осталось. Гейт на
последней вехе: **111 тестов зелёные**, `tsc` чист, `vite build` ок. Дальнейшая работа —
доработки/полировка, не достройка.

**Стек:** React 19 + Ant Design 6 + Zustand 5 + Vite 6 + TypeScript + Vitest 3. Node 24.

## Команды и гейт

```bash
npm install
npm run dev        # http://localhost:5173
npm run build      # tsc -b + vite build
npm test           # vitest run (весь набор)
npm run typecheck  # tsc --noEmit
npm run atlases    # пересборка спрайт-атласов (см. docs/assets.md)
```

**Гейт после любого содержательного изменения** (дисциплина проекта): `npm test` зелёный +
`npm run typecheck` чист + `npm run build` ок. Для UI-правок — живой прогон (Playwright MCP):
пользователь сам не тестирует, нужен самостоятельно проверенный результат.

## Конвенции работы

- **Оркестратор + субагенты.** Крупные куски по чёткой спеке делегируй субагентам (по задаче
  на агента), чтобы не раздувать контекст. Прогресс веди в `PROGRESS.md` на диске.
- **Инвариант слоёв (§2):** игровая логика — только в `src/core/*` (чистые функции, без импорта
  UI). UI (`src/ui`) не содержит правил игры. Правки редизайна не трогают ядро.
- **Гейт после каждого этапа**, не в конце. Не копи непроверенные изменения.

## Ключевые решения (зафиксированы)

- **Профиль баланса:** дефолт `development` (быстрый прогресс: вехи модов 5/5, дроп 10%,
  умение в магазине 50%). `production` (вехи 75/175, дроп 1%, 3%) включается
  `VITE_GAME_PROFILE=production` или `setProfile('production')`. Тесты §18.4 гоняются на
  production-порогах. Осознанно: prod-сборка стартует на dev-дефолте (см. code-review #7).
- **Балансовые формулы боя** (урон/защита/крит) — вне нормативного scope §0, наша линейная
  модель, документирована в [`docs/balance.md`](docs/balance.md).
- **Графика — атласы, генерятся в конце.** 5 AI-листов, нарезка кодом через CSS
  `background-position`. Манифест ассетов вытекает из заморожённой вёрстки, а не угадывается.
  Пользователь генерирует листы сам (ChatGPT), агент готовит промпты/нормализатор. Детали и
  грабли — [`docs/assets.md`](docs/assets.md).

## Прежде чем менять код — прочти инварианты

Раздел «Неочевидные инварианты» в [`docs/architecture.md`](docs/architecture.md): boss-механики
как флаги (не статусы), стихия урона ≠ `tags[0]`, `ctx.resist` только с ctx, действия над модами
через `campaign/mods.ts`, поражение не теряет прогресс, единый `resolveCardOutcome`. Эти вещи
молча ломаются при правках.
