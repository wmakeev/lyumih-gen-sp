# Ассеты и конвейер спрайт-атласов

Вся графика игры — 5 AI-сгенерированных спрайт-атласов, нарезаемых кодом через CSS
`background-position`. Отдельных файлов на юнит нет. Этот документ описывает конвейер
и грабли, которые легко наступить при перегенерации.

## Источник истины

`src/ui/assets/atlas-manifest.json` — единая точка истины для нарезки И для обёртки
`<Sprite>`. Определяет для каждого атласа: `src`, `cellW/cellH`, `cols/rows`, список `ids`.
**Порядок `ids` = порядок ячеек слева-направо, сверху-вниз** → детерминированный
`index → (row, col)`. Пока `src=null` — `<Sprite>` рендерит фолбэк-эмодзи.

Пять атласов:

| Атлас       | Ячейка | Сетка | Назначение |
|-------------|--------|-------|------------|
| `units`     | 128    | 6×4   | Мелкие боевые токены (20–44px). Светлый упрощённый арт, толстый костяной контур — читается на 40px. |
| `portraits` | 256    | 6×4   | Крупные детальные портреты (96–128px). **Те же `ids`, что у `units`** — один герой, два листа под свой размер. |
| `icons`     | 128    | 6×4   | 23 иконки: 9 статов + 11 CardKind + 3 слота экипировки. |
| `meta`      | 96     | 6×2   | 10 эмблем: оси Memento, группы модов, глифы хаба. |
| `tiles`     | 64     | 8×4   | Пол/стена/спавн + подсветки (в UI подключены только floor/wall). |

## Арт-дирекция

Стиль — силуэтный gothic-flat «Memento Mori» с одним акцентным цветом, минимум мелкой
детали (диффузия не держит мелочь и консистентность между листами). Палитра из
`src/ui/theme.ts`: акцент `#B0302B`, фон `#0E0D12`, кость `#EDE8DD`, тела токенов
`#322E3C–#4A4452`. Промпты под каждый лист — в `resources/prompts/` (`00-style-guide.md`
+ `01…05`). Блоки промптов на английском между `--- PROMPT START/END ---`; id/порядок/размеры
берутся из манифеста.

## Хромакей-фон (ключевое решение)

Фон исходных листов — **сплошной хромакей-зелёный `#00FF00`, НЕ «transparent»**. ChatGPT
физически не экспортирует alpha: на запрос «transparent» он запекает в пиксели двухтоновую
шахматку, враждебную вырезу (кость ≈237 сливается с белым фоном ≈250). Зелёного в палитре
арта нет → вырез в коде = один порог без потерь. Проверенный приём генерации: image-edit
«замени фон на `#00FF00`, арт не трогай».

CSS не умеет хромакеить зелёный в прозрачность, поэтому нужен **однократный бейк**
исходник→чистый прозрачный лист. Это не «нарезка на файлы» — рендер остаётся CSS-офсетным
из одного листа.

## Нормализатор: `npm run atlases`

`scripts/normalize-atlases.mjs` (sharp). Конвейер на каждую ячейку:
`extract по сетке манифеста (+inset — срезать белые линии сетки) → хромакей-вырез
(метрика `g − max(r,b)`, LO40/HI110) + despill кромки → alpha-trim bbox → contain+pad
(спрайты) ИЛИ fill (тайлы-оверлеи) → композ в чистый лист cols·cellW × rows·cellH с alpha`.
Выход → `public/atlases/<name>.png`, затем проставляет `src` в манифест. **Идемпотентен.**

Вход — `resources/atlases/0X-*.png` (имена по промптам). Выход `public/atlases/` коммитится
(его раздаёт Vite как `/atlases/<name>.png` и он попадает в `dist/`).

## Грабли при ПЕРЕгенерации листа (обязательно проверить)

- **Портреты — ряды НЕравномерны.** AI кладёт линии-разделители рядов не по 256
  (наблюдалось `y=0/272/523/766/1023`). Равномерная сетка → белая линия сверху + срезанные
  ноги. Лечится опцией `rowLines`(длина `rows+1`)+`rowInset` у источника: ряды режутся по
  фактическим линиям, колонки остаются равномерными. **Переизмерить `rowLines` при
  перегенерации листа портретов.**
- **Тайлы — неравномерная раскладка.** 7 фигур разложены не по 8-колоночной сетке манифеста,
  часть полые (рамка/уголки) → авто-сетка их режет. Заданы явные `srcBoxes` (измерены
  детекцией контента). **Переизмерить `srcBoxes` при перегенерации листа тайлов.**
- Прочие листы (`units`/`icons`/`meta`) — сетка равномерная, `inset` дорезает линии.

Требование к арту: каждый спрайт — отдельный «остров», окружённый зелёным, ничего не
пересекает линии сетки. Линии реза в картинке не рисуем — рез детерминирован манифестом.

## Интеграция в UI

- `<Sprite atlas id size>` (`src/ui/components/Sprite.tsx`) — выводит ячейку через
  `background-position`; общий экспорт `atlasSlice(atlas, id, size)` переиспользуется.
- `<Portrait>` (`src/ui/components/Portrait.tsx`) — крупная костяная рамка; спрайт по тому
  же id, что боевой токен (classId героя / raceId врага). **Грабля:** фон рамки держать
  тёмным (ниша), акцент инстанса — только в цвет рамки; заливка фона акцентом просвечивает
  сквозь прозрачный фон спрайта = цветной ящик.
- Тайлы поля (`floor`/`wall`) — фон клетки в `BattleScreen` (`BattleGrid`); подсветки
  вынесены в оверлей `::after` поверх тайла, токену дан `z-index:1`.

## Что не подключено (опционально)

Тайлы `spawn` и `hl_*` не используются — подсветки поля остались CSS-паттернами (сделаны так
намеренно ради дальтонизма). Атлас `tiles` их содержит, но террейн знает только floor/wall.
