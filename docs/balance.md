# Балансовая модель боя

Спека (§0) выносит «балансные числа урона» из scope (кроме нормативных формул Memento §16). Ниже — наша линейная модель, реализованная в `src/core/battle/damage.ts` и `engine.ts`.

## Величина умения (§6.6)

```
amount = skillFlat + resolvePercentValue(level, scaleToken) + statBonus
statBonus = round(caster.stats[card.statSource])     // прямой вклад стата, K=1
```

`level` — L носителя на момент замаха (+ `cardLevelBonusPerItemLevel` от экипировки, Этап 3). Для `strike` level = itemLevel оружия (или 0 для «кулаков»).

## Применение модов носителя (§16.12)

1. База (выше).
2. `damageMult` / `healMult` модов: `amount × (1 + Σ effective/100)`.
3. Крит: шанс = `critChance + Σ crit_chance_add`; при срабатывании `× 1.5`.
4. Защита (только урон): `final = max(1, amount − target.defense)`.
5. Проки `proc_extra_hit`: независимые RNG, каждый доп. удар считается заново.
6. Лайфстил/самолечение/отражение — после нанесения, от фактического урона.

## Прочее

- **speed** = число клеток перемещения за ход (BFS, §6.4).
- **initiative** = ключ сортировки очереди (убывание); тай-брейк по `id` юнита (детерминизм).
- **Defense** — плоское вычитание, минимум урона 1 (чтобы атаки не обнулялись полностью).
- **Crit multiplier** = 1.5 (константа `CRIT_MULTIPLIER`).
- **Резисты по расам/тегам** (§13.3) — заглушка `ctx.resist`, подключается на Этапе 3.
- **Ранний коэффициент силы врагов** (`spawn.ts`): `ENEMY_POWER_SCALE = 0.5`, `BOSS_POWER_SCALE = 0.7`. Контент-заглушки задали высокие `baseStats`, из-за чего стартовый отряд не пробивал ранних врагов (проверено живым прогоном). Множитель делает старт проходимым; с ростом `worldPower` враги усиливаются по §5.2/§13.2. Чистый балансовый рычаг — настраивается одной константой.

Все числа собраны в одном месте и легко перенастраиваются без изменения логики.
