# Журнал находок и граблей (опыт разработки)

Сквозной версионируемый леджер опыта: **находки** (как что-то реально устроено),
**решения с контекстом** (почему сделали так, а не иначе) и **грабли** (ошибки,
на которые уже наступали — чтобы не повторить). Живёт в репо, версионируется git,
переживает суммаризацию контекста и смену сессий.

## Отношение к другим докам (без дублей)

- [`PROGRESS.md`](../PROGRESS.md) — **хронология** «что сделали по вехам».
  Здесь — **сквозные уроки**, которые переживают вехи и полезны вне своего этапа.
- [`docs/architecture.md`](architecture.md) §«Неочевидные инварианты» — короткий
  **чек-лист** «что молча ломается». Сюда пишем разбор **почему**; туда — одну
  строку-инвариант.
- Доменные грабли остаются в профильном доке (напр. [`docs/assets.md`](assets.md)
  §«Грабли перегенерации»); здесь — короткий указатель на них.

## Дисциплина (когда читать / когда писать)

- **Перед** работой в незнакомой области — `grep` по этому файлу + по §«Неочевидные
  инварианты» в architecture.md. Дёшево, экономит час на уже пройденных граблях.
- **После** того как: (а) починил неочевидный баг, (б) выяснил неочевидное
  поведение, (в) принял решение, которое из кода не вывести — **добавь запись**.
  Правило: если это удивило тебя сейчас — удивит и следующего.
- Куда ещё продублировать: находку-инвариант (легко нарушить кодом) → одна строка
  в architecture.md; доменную грабля → профильный док. Журнал остаётся индексом.
- Формат — шаблон ниже. Свежие записи сверху. Дата — абсолютная (`ГГГГ-ММ-ДД`).

### Шаблон записи

```
### ГГГГ-ММ-ДД — <короткий заголовок> `[area:тег]`
**Симптом:** что выглядело сломанным / что было неочевидно.
**Причина:** корневая причина.
**Решение:** что сделали (+ коммит/файл/§spec).
**Не повторить:** конкретный guard — как впредь не наступить (инструмент, проверка, инвариант).
```

## Записи

### 2026-07-10 — Лист `units` резал ноги фигур на поле боя `[area:atlases]`
**Симптом:** токены персонажей/врагов на поле боя обрезаны снизу — не видно ног и
нижней кромки костяной обводки (портреты, т.е. более крупный лист, чинили раньше —
а `units` остался).
**Причина:** нормализатор брал лист `01-units-tokens.png` по **равномерной сетке
6×4** манифеста, но AI разложил 16 токенов со сдвигом по **обеим осям** (ряды
`y≈49–275/318–529/568–774` пересекают границы 256/512/768; колонки тоже смещены).
Окно ячейки с `inset` отсекало контент, вышедший за равномерную границу → `alpha-trim`
не мог вернуть уже отрезанное. Та же болезнь, что у портретов (лечилась `rowLines`,
коммит `1e64801`), но `units` числился «равномерным» и мимо неё прошёл.
**Решение:** заданы явные `srcBoxes` на каждый из 16 спрайтов = пересечение
фактических content-полос по X и Y (полосы разделены зелёными коридорами ≥12px, так
что соседи не попадают). Атлас пересобран `npm run atlases`. Коммит `4f36f3c`,
`scripts/normalize-atlases.mjs`. Обновлён `docs/assets.md`.
**Не повторить:** новый/перегенерённый AI-лист **не доверять** равномерной сетке
манифеста вслепую. Сперва — `npm run atlas:bands -- <файл> <cols> <rows>`
(`scripts/measure-atlas-bands.mjs`): он показывает фактические полосы контента и
сверяет с сеткой. При сдвиге/пересечении границ — `srcBoxes` (неравномерно по обеим
осям) или `rowLines` (неравномерны только ряды).
