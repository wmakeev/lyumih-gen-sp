# План редизайна Gen

> Источник стратегии — `game-deep-research/report.md` (deep-research, дата 2026-06-28).
> Источник состояния кода — `PROGRESS.md` (этапы 1–3 завершены, UI работает на эмодзи + дефолтная тёмная тема AntD).
> Память: `gen-project-decisions.md` — графика генерируется **в самом конце**, после редизайна вёрстки на плейсхолдерах.

## 0. Цель и рамки

**Что есть сейчас.** Полностью рабочая игра (75 тестов зелёные, `vite build` ок, smoke пройден). UI — `src/ui/` на React 19 + AntD 6 + Zustand. Тема минимальная: `main.tsx` задаёт лишь `darkAlgorithm` + `colorPrimary: '#7c5cff'`. Вся иконографика — эмодзи: статы из `STAT_META` (`src/core/types/stats.ts`), классы/расы/предметы — поля `emoji`/`iconEmoji` в `src/core/content/*` и `src/ui/components/UnitIcon.tsx`.

**Что делаем.** Превратить «дефолтный AntD на эмодзи» в осмысленный визуальный продукт в теме *Memento Mori*: тёмный силуэтный gothic-flat с одним акцентным цветом. Делаем это **в порядке возрастания риска** — сперва тема (нулевой риск), затем структурные UX-улучшения боя/меты, затем слой плейсхолдеров под спрайты, и **только в самом конце** — генерация атласов и точечная замена эмодзи.

**Чего НЕ делаем.** Не трогаем ядро (`src/core/battle`, `src/core/memento`, `src/core/campaign`) и баланс — редизайн чисто презентационный. Любое изменение типов контента (добавление `atlasId`) — аддитивное, без удаления `emoji`-полей (эмодзи остаётся фолбэком).

**Инвариант плана:** после каждого этапа — зелёные тесты (`vitest run`), чистый `tsc --noEmit`, успешная `vite build`. Графика — последней, чтобы манифест ассетов (id + точные пиксельные размеры) вытекал из готовой вёрстки, а не угадывался.

## Этап A — Тема AntD «Memento Mori» (нулевой риск ассетов)

**Зачем первым:** мгновенный визуальный эффект, не зависит ни от какой графики, не ломает логику. (Отчёт §6, §9.1.)

### A1. Вынести тему в отдельный модуль
- Создать `src/ui/theme.ts` — экспортирует объект `ThemeConfig` (тип из `antd`). Сейчас тема захардкожена инлайном в `main.tsx:15` — вынести, чтобы её можно было расширять и переиспользовать.
- `main.tsx` импортирует `gameTheme` и передаёт в `ConfigProvider`.

### A2. Seed-палитра Memento Mori
Опорные токены (точные hex подобрать на контраст-чекере — см. этап F):
- `algorithm: [theme.darkAlgorithm, theme.compactAlgorithm]` — тёмная + компактная под высокую инфо-плотность (9 статов, статусы, кулдауны, бейджи очереди). (Отчёт §6.2.)
- `colorPrimary` — акцент Memento: багровый/костяной (старт `#8B0000` либо приглушённый кровавый; финал — по контрасту). Один акцентный цвет — ключ консистентности с будущими спрайтами (§3).
- `colorBgBase: '#0E0D12'` (глубокий тёмный фон), `borderRadius: 4` (готик-флэт — малые радиусы).
- `fontFamily` с гарантированной кириллицей: `Inter` / `PT Sans` (НЕ декоративный фэнтези-шрифт в корпусе — многие не имеют кириллицы и ломают русский UI). Декоративный шрифт — только для крупных заголовков/логотипа, отдельно. (§6.6.)

### A3. Component Tokens (точечно, не задевая соседей)
Через `components: { … }` в теме (§6.3):
- `Card` — рамки/фон под «карточный» вид (Таверна, Персонаж).
- `Tooltip` — плотные тултипы под инфо-плотность.
- `Button` — акцент для боевых действий.
- `Tabs` — хаб уже вкладочный, согласовать с темой.

### A4. Подключить шрифты
- Self-host (`@fontsource/inter` / `@fontsource/pt-sans`) либо `<link>` в `index.html`. Self-host предпочтительнее (офлайн, нет зависимости от CDN). Подключить подмножество с кириллицей.

### A5. Кастомный слой читает токены темы, не хардкодит цвета *(архитектурная правка)*
**Проблема:** стилизация ручных компонентов сейчас — инлайновые JS-объекты с захардкоженными цветами (напр. `UnitIcon.tsx`: `background`, `border` строками). При смене палитры Memento они не подхватятся → цвет придётся править в двух местах.
**Решение:** AntD 6 по умолчанию `cssVar` (отчёт §6.4) → ручные зоны (грид боя, токены юнитов, рамки в стиле ассетов) писать обычным CSS, потребляющим **те же CSS-переменные темы** (`var(--ant-color-*)`).
- Завести `src/ui/theme.css` (или CSS-модули) для ручного слоя; инлайновые цветовые литералы вынести в классы поверх `var(--ant-*)`.
- Геометрия/размеры могут оставаться в пропсах (`size` у `<Sprite>`), но цвет/фон/бордер — через токены.
- Это снимает двойной источник правды по цвету и делает смену акцента Memento одноточечной (этап F правит только Seed-токены).

**Гейт A:** визуальный прогон (Playwright smoke), все экраны рендерятся в новой теме, `tsc`/`vitest`/`build` зелёные. Эмодзи пока на месте — меняется только хром/палитра/типографика. Смена `colorPrimary` в `theme.ts` визуально пробрасывается и в ручной слой (нет хардкод-цветов).

## Этап B — UX боевого экрана: «перфектная информация»

**Зачем:** главный verified-вывод отчёта — в плотном тактическом UI выигрывает не «красота», а **телеграфирование** (§2.1, §1.5). Это структурные правки `src/ui/BattleScreen.tsx`, независимые от спрайтов.

### B1. Превью исхода действия до коммита (ядро принципа)
- При наведении/выборе цели для атаки/умения — показывать предполагаемый **урон/попадание** и подсветку поражаемых клеток (для aoe — всю зону) **до подтверждения**. Расчёт — переиспользовать чистые функции `src/core/battle/damage.ts` (без мутаций состояния).
- Это снимает основную боль «дефолтного» боя: исход хода виден заранее, ход играется как детерминированный паззл. (ITB-канон, §2.1.)

### B2. Однозначная маркировка сетки
- Подсветки move / range / LoS / target — различать **паттерном/обводкой/иконкой, а не только оттенком** (дальтонизм; §8.1). Например: перемещение — заливка, дальность атаки — обводка, цель — рамка-маркер.
- Свой / враг / downed — дублировать форму/рамку/иконкой, не только цветом. Downed — иконка + опрозрачнивание + перечёркивание (`UnitIcon` уже умеет `dimmed`, расширить).
- Бейджи порядка хода на клетках — фикс-размер слот (готовим под атлас 5).

### B3. Очередь хода и панель действий
- Очередь по инициативе — явные бейджи порядка, активный юнит выделен. (§2.1.)
- Панель действий активного юнита: умения + кулдауны (числом на иконке), базовая атака, move, end turn — собрать в устойчивый блок фикс-размеров (готовим слоты под атлас 3).

**Гейт B:** Playwright smoke боя — превью исхода видно до клика, состояния клеток различимы без опоры на цвет, тесты/tsc/build зелёные. (Ядро не трогаем — правки только в `src/ui/`.)

## Этап C — Мета-экраны: раздельные панели Memento Mori

**Зачем:** verified-паттерн Hades — мета-функции разносятся по отдельным выраженным панелям, со слотами «занято/свободно» (§2.2, §7.4).

### C1. Выраженная панель Memento Mori
- Оси силы (смерть / использование / победа) — прогресс-бары + иконки осей (слоты под атлас 4). Данные уже есть в campaign-слое.
- Хаб уже вкладочный (`HubScreen` — 6 вкладок) — это согласуется с «раздельными панелями»; Memento заслуживает собственной выраженной зоны/вкладки.

### C2. Слоты модов со слотовой визуализацией
- Слоты носителя: занятый = иконка мода + бейдж «M+»; пустой = пунктир-плейсхолдер. Грид доступных модов — иконки фикс-размера. (§7.4.)
- UI обязан **показывать вклад** мета-прогресса, не пряча его (баланс — за геймдизайном, не трогаем).

**Гейт C:** Memento/моды отрисованы раздельными панелями со слотами; smoke по вкладкам; тесты/tsc/build зелёные.

## Этап D — Слой плейсхолдеров `<Sprite>` (мост к спрайтам)

**Зачем:** единая точка переключения «эмодзи → спрайт», чтобы вёрстка была готова к ассетам ДО их генерации и не поехала при замене (§7, §9.2). **Это последний этап перед заморозкой раскладки.**

### D1. Компонент-обёртка `<Sprite>`
- `src/ui/components/Sprite.tsx`: пропсы `atlas`, `id`, `size` (плюс `dimmed`, `title`, `accent` — миграция API `UnitIcon`).
- Поведение: если для `(atlas,id)` нет нарезанного спрайта в манифесте → рендерит **текущий эмодзи** в контейнере **точного целевого размера** (фикс aspect-ratio). Когда атлас появится → рендерит `background: url(atlas) -X -Y`.
- Dev-режим: плейсхолдер может показывать имя ассета + ожидаемый размер (для ревью раскладки).

### D2. Перенести «идентичность иконки» из ядра в UI *(архитектурная правка)*
**Проблема:** презентация протекла в домен — `STAT_META.emoji` лежит в `src/core/types/stats.ts`, `iconEmoji`/`emoji` — в `src/core/content/*`. В мире атласов «какая картинка у warrior» захочет жить в UI-манифесте. Оставить как есть = **два источника истины** (emoji в ядре + atlas-манифест в UI), которые разъедутся.
**Решение — единый источник истины на стороне UI:**
- Ядро отдаёт только **семантический id** (`statId`, классы/расы/предметы — их строковые id). Отображение целиком в UI.
- Завести UI-таблицу `src/ui/assets/icon-registry.ts`: `id → { emoji, atlas, slot? }` — фолбэк-эмодзи живёт здесь, не в `core`.
- Эмодзи переносятся из `core/types/stats.ts` и `core/content/*` в этот реестр. Если поле `emoji` в контенте больше нигде не читается — удалить его из core-типов; если читается тестами/иным кодом — оставить помеченным `@deprecated`, но UI на него не опирается.
- `STAT_META` оставляет в ядре только семантику (`ru`-подпись), эмодзи уезжает в реестр.

### D3. JSON-манифест атласов — единая точка истины
- `src/ui/assets/atlas-manifest.json`: `atlasId → { cellW, cellH, cols, rows, ids: [...] }`. Один файл для нарезки И для обёртки (§5.3, §7.5).
- Маппинг `id → (atlas,row,col)` детерминированный (порядок ids = порядок ячеек слева-направо, сверху-вниз).
- Реестр иконок (D2) и манифест атласов согласованы по `id`: реестр = «что показать», манифест = «где это в атласе».

### D4. Перевести существующие точки эмодзи на `<Sprite>`
Заменить прямой рендер эмодзи на `<Sprite>` (читает реестр D2 + манифест D3) в:
- `src/ui/components/UnitIcon.tsx` → проксирует на `<Sprite atlas="units">` (либо `UnitIcon` поглощается `Sprite`).
- `src/ui/components/StatStrip.tsx` → `<Sprite atlas="icons" id={statId}>` вместо `meta.emoji`.
- `BattleScreen.tsx`, `InterBattleScreen.tsx`, `ExpeditionTab.tsx`, `PersonaTab.tsx`, `TavernTab.tsx` — точки из грепа `grep -rn emoji src/ui`.
- Источник отображения — **только реестр иконок** (D2), не поля `core`. Фолбэк-эмодзи берётся из реестра.

**Гейт D:** все экраны выглядят идентично этапу C (эмодзи через `<Sprite>`-фолбэк из UI-реестра), но каждый ассет-слот теперь фикс-размера, читает манифест, а `core` не содержит презентационных эмодзи. Тесты/tsc/build зелёные. **После этого вёрстка заморожена → точные размеры ячеек известны.**

## Этап E — Генерация атласов и точечная замена (последним)

**Зачем последним:** манифест размеров вытекает из готовой вёрстки (D). Главный verified-риск: диффузия не держит мелкую деталь и консистентность между листами → стиль с минимумом мелочи, однотипное на одном листе. Бюджет 3–5 генераций. (§3, §5, память.)

### E1. Зафиксировать раскладку из реальных размеров
Снять фактические `size` из заморожённой вёрстки (D) и заполнить манифест. Целевая раскладка (§5.1), приоритет при урезании до 3 листов (§5.2):
1. **Лист 1 — токены юнитов** (8 классов + 8 рас, ~256×256, грид 6×4) — без них поле боя нечитаемо.
2. **Лист 3 — иконки статов + CardKind + слоты** (9+11+3=23, ~128×128, 6×4) — вся плотная информация UI.
3. **Лист 5 — тайлы поля + подсветки** (~64–128px, 8×4) — читаемость сетки.
Догенерация в рамках бюджета: **Лист 2** боссы (painterly допустим, ~512×512), **Лист 4** склонности/моды/статусы.

### E2. Промпты (силуэтный gothic-flat)
Один промпт = один лист = один тип элементов в едином масштабе (§3.2). В каждом промпте явно: стиль `flat`, `bold silhouette`, `thick outline`, `no gradients`; палитра словами + hex-якоря (один акцент); `centered in cell`, `uniform size`, `transparent background`, `no drop shadow outside cell`, `no bleeding into neighbors`; точная сетка N×M; перечисление элементов по позициям (для детерминированного index→id).

### E3. Нарезка и нормализация
- Слайсер fixed-stride: `cell = (col·cellW, row·cellH)`, Grid-режим (НЕ MaxRects — лист уже почти сетка). (§5.)
- Padding ≥2px заложен в промпт, но **AI не гарантирует pixel-perfect** → обязательный пост-нормализатор: alpha-trim каждого спрайта + ресайз в фикс-ячейку. (§5.3, риск §10.2.)
- Скрипт нарезки — Node-утилита (напр. `sharp`), вход = PNG-лист + манифест, выход = либо координаты в манифесте для CSS `background-position`, либо отдельные PNG. Грузить атласы **отдельными ресурсами**, не инлайнить в JS-чанк (текущий бандл 892 КБ; §8.2).

### E4. Точечно включить спрайты
- По мере готовности листов переключать `<Sprite>` с эмодзи на атлас, экран за экраном: Battle → Hub → Таверна → Memento. Фолбэк-эмодзи остаётся для незакрытых id.

**Гейт E:** каждый включённый лист проходит визуальный smoke; нет «битых» слотов (фолбэк ловит пропуски); вес атласов измерен.

## Этап F — Доступность и перфоманс (проверка, не угадывание)

> Отчёт помечает §8 как `[⚠ gap]` — не покрыто verified-источниками. Это **проверка инструментами**, а не вкусовщина.

- **Контраст:** прогнать палитру Memento контраст-чекером, цель WCAG AA — текст ≥4.5:1, иконки/крупный текст ≥3:1 на тёмном фоне (тёмная тема легко проваливает приглушённые акценты). Скорректировать hex этапа A2. (§8.1.)
- **Дальтонизм:** прогнать симулятором (дейтеранопия) — убедиться, что свой/враг/downed и подсветки различимы без опоры на цвет (закрыто структурно в B2). (§8.1.)
- **Перфоманс:** замерить вес атласов, сжать PNG (`pngquant`/`oxipng`); Vite code-splitting по вкладкам/роутам, lazy-load фоновых атласов (боссы, фоны). Сравнить бандл до/после. (§8.2.)

**Гейт F (финал):** контраст AA подтверждён, дальтонизм-симуляция чистая, бандл измерен и не деградировал, полный Playwright smoke зелёный.

## Сводка порядка (по риску) и зависимости

| Этап | Содержание | Риск | Зависит от |
|---|---|---|---|
| A | Тема AntD Memento (токены, шрифты) | нулевой | — |
| B | UX боя: превью исхода, маркировка сетки | низкий | A |
| C | Раздельные панели Memento + слоты модов | низкий | A |
| D | Слой `<Sprite>` + JSON-манифест, миграция эмодзи | низкий | A,B,C |
| E | Генерация атласов → нарезка → замена | **высокий** (AI) | D (заморозка размеров) |
| F | Контраст/дальтонизм/перфоманс — проверка | низкий | A,E |

**Принцип:** A→D — чистая фронт-работа без графики, каждый этап самодостаточен и проверяем. E — единственный AI-зависимый, идёт последним на заморожённой вёрстке. F прослаивает: контраст-чек палитры можно сделать сразу после A, перф-замер — после E.

## Открытые вопросы (проверять прототипом, не принимать как данность)
1. Арт-стиль под «1 промпт = 1 атлас» — verified-сравнения нет, рекомендация силуэтного gothic-flat выведена из failure-mode; **проверить пробным листом в 2–3 стилях** перед массовой генерацией (§10.1).
2. Контраст/дальтонизм тёмной палитры и читаемость токенов 44px — этап F.
3. Эффект атласов на бандл (892 КБ) vs отдельные PNG при code-splitting — измерить (§10.1.3).
4. Конкретные deprecations AntD 6.x в компонентах Gen (smoke уже шумит про `Space direction`, `List`) — **свериться с актуальным `ant.design/docs/react/migration-v6`** перед рефактором хрома на этапе A (§6.7).

## Watch-items (осознанно НЕ трогаем в этом редизайне)
- **Мутация `BattleState` «на месте» + ручной `rev`-счётчик.** `engine.ts` мутирует состояние напрямую (`state.round += 1`, `applyDamage` пишет в `state`), стор дёргает `rev: s.rev + 1` для перерисовки React. Это смелл (уже ловили в PROGRESS), но редизайн **не добавляет доменных мутаций** — только презентационное/селекшн-состояние, которое в сторе уже иммутабельно (`{ ...s.ui }`). Профит от перехода на immer/иммутабельность сейчас ≈ ноль, риск дестабилизировать зелёное ядро реальный. Переходить — только если позже понадобятся доменные анимации/откаты состояния боя.
- **Превью исхода (этап B) ядро НЕ меняет:** `baseCardAmount`/`resolveCardAmount`/`statBonus` уже чистые функции — «сухой» расчёт без мутаций возможен переиспользованием как есть (тот же путь, что у greedy-AI). Изменения в `damage.ts` не требуются — зафиксировано как проверенный факт, не задача.
