# Промпты генерации атласов Gen — общий гайд

Эти промпты — для генерации **спрайт-атласов** в ChatGPT (GPT image / DALL·E).
Источники: `game-deep-research/report.md` (§3 арт-стиль, §5 упаковка), `redesign-plan.md`
(этап E), и **главный источник истины — `src/ui/assets/atlas-manifest.json`** (заморожённая
вёрстка: id, порядок ячеек, размеры).

## Что генерируем (4 листа — ровно как в манифесте)

| Файл | Атлас | Сетка | Ячейка | Элементов | Стиль |
|---|---|---|---|---|---|
| `01-units-tokens.md` | `units` | 6×4 (24) | 128×128 | 16 | **мелкие токены 20–44px**: упрощённый, светлый, толстый контур |
| `02-icons.md` | `icons` | 6×4 (24) | 128×128 | 23 | flat-vector иконки статов/умений/слотов |
| `03-meta.md` | `meta`  | 6×2 (12) | 96×96  | 10 | flat-vector эмблемы осей/модов/глифов |

> Лист «боссы» (§5.1 отчёта) в код **не заморожен** — генерируем позже, если понадобится.

## Единая арт-дирекция (Memento Mori) — общая для всех листов

- **Стиль:** dark gothic **flat** vector, **bold silhouette**, **thick uniform outline**,
  **no gradients**, **no rendering/3D**, minimal inner detail. Чем меньше мелкой уникальной
  детали — тем выше консистентность между листами (verified failure-mode диффузии, отчёт §1/§3).
- **Палитра (жёсткая, словами + hex-якоря из `theme.ts`):**
  - фон элементов/силуэт: глубокий near-black с фиолетовым подтоном — `#0E0D12`, `#17151E`
  - костяной/тёплый светлый (контур, блики, кость): `#EDE8DD`, `#C9C2B2`
  - **единственный акцентный цвет — приглушённый кровавый/гранатовый `#B0302B`** (светлее — `#D7615A`)
  - допустим один холодный полутон для нежити/призраков (бледно-голубой `#6E7E8C`) — но скупо
- **Один акцент на всё.** Акцент `#B0302B` — единственный «цветной» элемент; всё остальное в
  костяно-чёрной гамме. Это держит консистентность между листами на палитре, а не на «памяти» модели.

## Глобальные правила упаковки (в каждом промпте повторены)

- **Точная сетка N колонок × M рядов**, **uniform cell size**, элементы **centered in cell**,
  **equal margins**, **solid flat chroma-green `#00FF00` background** (НЕ шахматка, НЕ белый — см. ниже).
- **No drop shadow outside the cell. No element bleeding into neighbouring cells.** Padding между
  элементами ≥ соответствует «воздуху» вокруг (нарезка fixed-stride в коде).
- **Один стиль/палитра/масштаб на лист.** Не смешивать масштабы (портреты ≠ иконки).
- Элементы перечислены **по позициям слева-направо, сверху-вниз** — порядок строго совпадает с
  `ids` в манифесте, чтобы `index→(row,col)→id` мапился детерминированно.

## Фон = ХРОМАКЕЙ, а НЕ «прозрачность» (критично — проверено на практике)

**ChatGPT физически не экспортирует alpha-канал.** Когда просишь «transparent background»,
он печёт в пиксели **двухтоновую шахматку**, имитирующую прозрачность. Эта шахматка враждебна
вырезу: чисто выбить её кодом не выходит (осыпь белых точек + каёмка по краю костяных деталей,
т.к. кость `#EDE8DD≈237` почти сливается с фоном `≈250`). Поэтому **прозрачность у модели не просим.**

**Вместо этого просим сплошной хромакейный фон, которого нет в палитре арта:**

- **Сплошной равномерный хромакей-зелёный `#00FF00`** на весь холст за всеми ячейками.
  В палитре арта (near-black / кость / кровавый) зелёного нет вообще → вырез в коде = один
  порог по цвету, **без потерь, без flood-fill, без осыпи**.
- Явно запретить шахматку/паттерн прозрачности/белый фон: `solid flat #00FF00, NOT a checkerboard,
  NOT a transparency pattern, NOT white`.
- Flat-арт с толстым тёмным контуром идеален под хромакей — контур поглощает каёмку, спилла нет.
- Полупрозрачные оверлеи (подсветки тайлов) рисуем **сплошным цветом** на зелёном; частичную
  прозрачность накладываем в коде через CSS-opacity (на зелёном фоне alpha не передать).

## Прочие оговорки

1. **Pixel-perfect сетку модель не гарантирует — и это НЕ нужно.** В коде стоит пост-нормализатор
   (хромакей-вырез → нарезка по сетке → **alpha-trim каждого спрайта** → ресайз/центрирование в
   фикс-ячейку, план E3). Он находит реальный bounding box каждого спрайта по альфе, поэтому точные
   отступы модели не важны. **Единственное жёсткое требование к модели для точной нарезки:** каждый
   элемент — **отдельный «остров», со всех сторон окружённый зелёным фоном**, ничего не пересекает
   линии сетки (оружие/крылья/рога не залезают в соседнюю ячейку). Тогда trim даёт чистый рез.
   Рисовать в картинке линии-направляющие НЕ просим — модель рисует их криво, и их потом удалять.
2. **Соотношение сторон холста** указано в каждом промпте (чтобы ячейки вышли квадратными). Точный
   размер не критичен — режется по сетке.
3. **Проверь пробным листом 2–3 стиля** (открытый вопрос §10.1 отчёта) перед массовой генерацией —
   начни с `01-units.md`, это ядро читаемости поля боя.
4. **Альтернатива перегенерации:** скормить готовый лист в режиме image-edit с просьбой «заменить
   фон на сплошной `#00FF00`, остальное не трогать» — сохраняет уже удачный арт, если модель не плывёт.

## Как использовать

1. Открой нужный `NN-*.md`, скопируй блок между `--- PROMPT START ---` и `--- PROMPT END ---`.
2. Вставь в ChatGPT (режим генерации изображений), приложи при желании уже сгенерированный
   предыдущий лист как style-reference для консистентности.
3. Сохрани PNG в `resources/atlases/<atlas>.png` (или куда настроен слайсер), прогони нормализатор,
   пропиши `src` в `atlas-manifest.json`.
